Requirements

Functional Requirements

##### Description
FR-1 Users have the ability to use their VT email to create an account. Verification will be handled via email. User has to create a password.
FR-2 Existing users must login with correct credentials (username and password).
FR-3 Users can access the system remotely.
FR-4 Authenticated users can view the campus population map.
FR-5 Users with privileges will be able to see real data which is at most five minutes behind real time.
FR-6 Users will be able to change time periods of the visualization.
FR-7 The map will be colored based on the population distribution across campus.
FR-8 Clicking on a building will drill into the 3D floor representation.
FR-9 Users can change basemap (topographic view, black and white map, Openstreet map, etc).
FR-10 Users can view the generated statistical summary to understand population density patterns.
FR-11 Users can set and get alerts (SMS & Email) when the population for a building reaches a threshold so that they can know the risk when they are not using the system.
FR-12 Users can get a daily report summarizing the population distribution over 24 hours so that they can make decisions on which buildings to clean and sanitize Users can print out the reports and maps I can generate a physical copy of the maps to share with concerned authorities

Non Functional Requirements

##### Description
NFR-1 Website will be available on all modern browsers.
NFR-2 Database will be scalable depending on volume.
NFR-3 Anonymity - Database will not contain any personally identifiable information.
NFR-4 Website and database will have 99%+ uptime - it will be Reliable.
NFR-5 Database and application will be up to date on security.
